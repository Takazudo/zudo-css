# Color Palette Panel

## The Problem

When generating UI components, you want to see how they look with different color palettes. Maybe the client wants to compare a blue brand versus a teal brand. Maybe you are exploring dark mode. Maybe the design team has proposed three palette options.

Without a live preview tool, the process is: edit CSS variables, save, wait for hot reload, look at the result, repeat. For every palette change, you repeat this cycle. With multiple components on screen, it is nearly impossible to get a holistic view of how a palette change affects the entire system — you end up tweaking one variable, refreshing, tweaking another, refreshing again.

This is slow, tedious, and discourages exploration. Designers and developers default to the first palette that "looks okay" instead of iterating toward the best option.

## The Solution

A **color palette panel** that exposes Tier 1 palette colors as live controls. When you adjust a palette color, every component on the page updates instantly — because theme tokens (Tier 2) reference palette tokens (Tier 1), and components reference theme tokens.

The panel does not change component code. It only changes the root-level palette CSS custom properties. The entire three-tier cascade does the rest.

## How It Works

The cascade flows in one direction:

```
Palette (Tier 1)  →  Theme (Tier 2)  →  Components (Tier 3)
--palette-blue-500    --theme-accent      button background
--palette-gray-900    --theme-fg          text color
--palette-gray-100    --theme-bg          page background
```

1. The **panel** exposes Tier 1 palette colors (e.g., `--palette-blue-500`, `--palette-gray-900`) as interactive color inputs
2. Components use Tier 2 **theme tokens** (`--theme-accent`, `--theme-fg`, `--theme-bg`)
3. Theme tokens reference palette tokens: `--theme-accent: var(--palette-blue-500)`
4. Changing a palette color in the panel cascades through the entire component set automatically

The button, card, and badge all consume theme tokens. None of them know about `--palette-blue-500` directly. When the panel changes that palette value, `--theme-accent` updates, and every component that uses `--theme-accent` re-renders with the new color — no component code changes needed.

## Panel Architecture

The color panel is a React component that reads the current palette values and provides controls to modify them. The architecture is straightforward:

```jsx
function hexToHsl(hex) {
  // Convert #rrggbb from <input type="color"> to hsl() string
  const r = parseInt(hex.slice(1, 3), 16) / 255;
  const g = parseInt(hex.slice(3, 5), 16) / 255;
  const b = parseInt(hex.slice(5, 7), 16) / 255;
  const max = Math.max(r, g, b), min = Math.min(r, g, b);
  const l = (max + min) / 2;
  if (max === min) return `hsl(0 0% ${Math.round(l * 100)}%)`;
  const d = max - min;
  const s = l > 0.5 ? d / (2 - max - min) : d / (max + min);
  let h = 0;
  if (max === r) h = ((g - b) / d + (g < b ? 6 : 0)) * 60;
  else if (max === g) h = ((b - r) / d + 2) * 60;
  else h = ((r - g) / d + 4) * 60;
  return `hsl(${Math.round(h)} ${Math.round(s * 100)}% ${Math.round(l * 100)}%)`;
}

function ColorPalettePanel({ paletteTokens }) {
  const handleChange = (tokenName, hexValue) => {
    // Convert hex from <input type="color"> to hsl() before setting
    const hslValue = hexToHsl(hexValue);
    document.documentElement.style.setProperty(tokenName, hslValue);
  };

  return (
    <div className="palette-panel">
      {paletteTokens.map((token) => (
        <div key={token.name}>
          <label>{token.name}</label>
          <input
            type="color"
            defaultValue={token.hexValue}
            onChange={(e) => handleChange(token.name, e.target.value)}
          />
        </div>
      ))}
    </div>
  );
}
```

Note that `<input type="color">` always returns hex values (`#3366cc`), so the handler converts to `hsl()` before setting the CSS custom property. This keeps the three-tier system consistently using `hsl()` values.

The key implementation detail: the panel sets CSS custom properties on `:root` using `document.documentElement.style.setProperty()`. This is a live DOM operation — the browser recalculates all styles that reference the changed property immediately. No React re-render of the components is needed; the CSS cascade handles propagation.

## Integration with Three-Tier Color

The panel is powerful _because_ of the [Three-Tier Color Strategy](/docs/color/three-tier-color-strategy). Without the three-tier separation, changing a color means finding every place it is used. With three tiers, the panel only needs to know about Tier 1 palette values — everything else follows.

Every component in this dashboard — sidebar, metric cards, data table, badges, status pills — inherits its color from the same palette. The color palette panel would let you swap the indigo palette for any other hue, and every element would update simultaneously. The sidebar background, the accent highlights, the badge colors, the table headers — all cascade from the same `--palette-indigo-*` root.

This is the core value: instead of manually updating colors in 15 different components, you change 3 palette variables and the entire dashboard transforms.

## When to Use

- **Design exploration** — Test multiple color palettes against real components to find the best fit before committing to a direction
- **Client presentations** — Show clients how their brand colors will look across the full UI, with live switching between options
- **Brand color iteration** — When the brand palette changes, validate the impact across all components in one view
- **Dark mode prototyping** — Swap the palette to dark values and instantly see how the entire component set adapts
- **Accessibility testing** — Quickly swap in higher-contrast palette values to check if the design system supports accessible color combinations
- **Design system development** — Validate that your three-tier color architecture actually works by proving that palette changes cascade correctly through all components

For the underlying color architecture, see [Three-Tier Color Strategy](/docs/color/three-tier-color-strategy). For the token system that governs spacing and sizing in the playground, see [Tight Token Strategy](/docs/methodology/tight-token-strategy).
