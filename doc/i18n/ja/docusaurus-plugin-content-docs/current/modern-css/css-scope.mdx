# CSS @scope（スコープ）

## 問題

DOMの特定のセクションにスタイルをスコープすることは、従来Shadow DOM（重くて柔軟性に欠ける）やBEMのような命名規則（冗長でエラーが起きやすい）が必要でした。コンポーネントベースのUIを構築する際、親コンポーネントのスタイルがネストされた子コンポーネントに漏れることがよくあります。AIエージェントが`@scope`を使うことは稀で、代わりにグローバルセレクターを生成したり、JavaScriptベースのスコーピングソリューションを提案したりします。

## 解決方法

`@scope`アットルールを使うと、スコープルート（スタイルの適用開始位置）とオプションのスコープリミット（スタイルの適用停止位置）を定義できます。これにより、ネストされたコンポーネントに影響を与えずに特定のDOMサブツリーをターゲットとする「ドーナツスコープ」のスタイルが作成されます。`@scope`はまた、カスケードにおける近接性ベースのスタイリングを導入し、詳細度が等しい場合に最も近いスコープルートが勝ちます。

## コード例

### 基本的なスコーピング

```css
@scope (.card) {
  /* These styles only apply inside .card elements */
  h2 {
    font-size: 1.25rem;
    margin-bottom: 0.5rem;
  }

  p {
    color: #4b5563;
    line-height: 1.6;
  }

  img {
    border-radius: 8px;
    width: 100%;
  }
}
```

```html
<div class="card">
  <h2>Card Title</h2>   <!-- Styled -->
  <p>Card content</p>    <!-- Styled -->
</div>

<h2>Page Title</h2>      <!-- NOT styled -->
<p>Page content</p>       <!-- NOT styled -->
```

### ドーナツスコープ

上限（スコープルート）と下限（スコープリミット）の両方を定義して「ドーナツ」を作成します — スタイルは外側の領域に適用されますが、内側の領域には適用されません。

```css
/* Style the card chrome but not the nested content area */
@scope (.card) to (.card-content) {
  /* These styles apply to .card's "chrome" only */
  padding: 1.5rem;
  border: 1px solid #e5e7eb;
  border-radius: 8px;

  header {
    font-weight: bold;
    margin-bottom: 1rem;
    border-bottom: 1px solid #e5e7eb;
    padding-bottom: 0.5rem;
  }
}
```

```html
<div class="card">
  <!-- Inside scope: styled -->
  <header>Card Header</header>

  <div class="card-content">
    <!-- Outside scope (limit reached): NOT styled -->
    <p>This content is not affected by the scoped styles</p>
    <header>Nested header is also not affected</header>
  </div>
</div>
```

### ネストされたコンポーネントへのスタイルの漏れを防止する

```css
/* Style .tabs but do not leak into nested .tabs components */
@scope (.tabs) to (.tabs) {
  :scope {
    display: flex;
    border-bottom: 2px solid #e5e7eb;
  }

  button {
    padding: 0.5rem 1rem;
    border: none;
    background: transparent;
    cursor: pointer;
  }

  button[aria-selected="true"] {
    border-bottom: 2px solid #2563eb;
    color: #2563eb;
  }
}
```

```html
<div class="tabs">
  <button aria-selected="true">Tab 1</button>
  <button>Tab 2</button>

  <div class="tab-panel">
    <!-- Nested .tabs has its own scope — parent styles do not leak -->
    <div class="tabs">
      <button aria-selected="true">Sub-Tab A</button>
      <button>Sub-Tab B</button>
    </div>
  </div>
</div>
```

### 近接性ベースのスタイリング

同じプロパティが同じ詳細度の2つの`@scope`ルールで設定された場合、最も近いスコープルートを持つものが勝ちます。

```css
@scope (.light-theme) {
  p {
    color: #1f2937;
  }
}

@scope (.dark-theme) {
  p {
    color: #f3f4f6;
  }
}
```

```html
<div class="light-theme">
  <p>Dark text (light-theme is nearest scope)</p>

  <div class="dark-theme">
    <p>Light text (dark-theme is nearest scope)</p>
  </div>
</div>
```

### スコープルートを参照する`:scope`の使用

```css
@scope (.alert) {
  :scope {
    padding: 1rem;
    border-radius: 4px;
    border: 1px solid;
  }

  p {
    margin: 0;
  }
}
```

### インライン`<style>`での`@scope`

`@scope`はインライン`<style>`要素内でセレクターなしで使用でき、自動的に親要素にスコープされます。

```html
<div class="widget">
  <style>
    @scope {
      /* Scoped to .widget, the parent of this <style> */
      p {
        color: green;
      }
    }
  </style>
  <p>This text is green</p>
</div>

<p>This text is NOT green</p>
```

## AIがよくやるミス

- `@scope`の存在を知らず、スタイルの漏れを防ぐためにBEMや複雑なセレクターを使用する
- `@scope`とShadow DOMのカプセル化を混同する — `@scope`は外部スタイルがスコープされた要素に適用されることを防がない
- ネストされたコンポーネントへのスタイルの漏れを防ぐためのドーナツスコープパターン（`to`キーワード）を忘れる
- 近接性ベースのスタイリングとカスケードとの相互作用を理解していない
- ネイティブの`@scope`で十分なユースケースに対して、JavaScriptベースのスコーピング（CSS modules、styled-components）を使用する
- `@scope`と`@layer`を混同する — レイヤーは優先度/詳細度の順序を管理し、スコープはスタイルが適用されるDOMサブツリーを管理する

## 使い分け

- 親スタイルがネストされた子コンポーネントに漏れてはいけないコンポーネントベースのスタイリング
- コンポーネントのラッパー/外枠はスタイリングし、コンテンツスロットには影響を与えないドーナツスコープ
- 近接性ベースのカスケード解決によるテーマ切り替え
- 直接のコンテキストのみに影響すべきインライン`<style>`ブロック
- `@layer`との併用 — 優先度管理にはレイヤーを、DOMターゲティングにはスコープを使用する
